# Проектная работа группы 2

## Описание проекта

API-сервис, разработанный на FastAPI для управления фильмами, жанрами и персонами, также включает аутентификацию пользователей.


## Установка и запуск

### Клонируем репозиторий

```Bash 
git@github.com:mishakras/notifications_sprint_1.git
cd Async_API_sprint_1_team_repo
```

### Создаем конфигурационные фалы из примеров

```Bash
cp admin_service/config/.env.example admin_service/config/.env
cp common/.env.example common/.env
```

### Собираем контейнеры:

```Bash
docker-compose build
```

### Запуск

```Bash
docker-compose up -d
```

Для остановки сервисов:
```Bash
docker-compose down
```

\* Вместо раздельных команд сборки и запуска используйте:
```Bash
docker-compose up -d --build
```

После запуска API будет доступно по адресу: http://localhost


## Структура проекта

`/admin_service` - код сервиса администрирования контента.
`/auth_service` - код сервиса авторизации.
`/content_service` - код сервиса выдачи контента.
`/frontend_service` - код сервиса фронтенда.
`/swagger_service` - код сервиса документации.
`/partitioner` - код сервиса партицирования таблиц PostgreSQL.
`/etl` - код скрипта ETL-процесса для сохранения данных в ElasticSearch.
`/tests` - тесты.

## Функциональные требования

### Лайки фильмов
   - Пользователь может поставить или снять лайк фильму.
   - Отображается общее количество лайков у фильма.

### Отзывы к фильмам
   - Пользователь может оставить, изменить или удалить текстовый отзыв.
   - Отображение списка отзывов к фильму.

### История просмотров
   - Автоматическое сохранение просмотренных фильмов.
   - Просмотр своей истории.
   - Возможность очистки истории.

### Закладки
   - Добавление и удаление фильма из закладок.
   - Просмотр списка закладок.

### Сбор аналитики пользовательского поведения
   - Сбор событий: недосмотренные фильмы, действия с плеером (переключение языков и др.).
   - Передача данных в аналитический модуль (например, через Kafka или очередь событий).

## Нефункциональные требования

### Производительность

- **RPS**: до `500` запросов в секунду (в пике).
- **DAU**: около `100,000` активных пользователей в сутки.
- **MAU**: около `2,000,000` активных пользователей в месяц.

### Объём данных

| Тип данных                   | Оценка объёма                        |
|------------------------------|--------------------------------------|
| Средний отзыв                | `512 байт – 1 КБ`                    |
| История просмотров           | `100–300 записей` на пользователя    |
| Закладки                     | `~50 записей` на пользователя        |
| События плеера               | `20–50` событий на один просмотр     |

### Хранение

- **Объём данных на одного пользователя**:
  - Отзывы + лайки: `~200 КБ`
  - История просмотров: `~100 КБ`
  - Закладки: `~20 КБ`
  - Поведенческая аналитика: `~1 МБ/мес`

- **Прогноз на 2 года**:
  ```text
  2 000 000 пользователей × 1.5 МБ ≈ 3 ТБ

### Масштабируемость

- Горизонтальное масштабирование.
- Кеширование часто запрашиваемых данных (например, Redis).

### Доступность и отказоустойчивость

- **SLA**: 99.9%
- Асинхронное логирование всех действий.
- Репликация и бэкапы (PostgreSQL или аналог).

### Безопасность

- Все действия требуют аутентификации.
- Только автор может редактировать/удалять свои отзывы.
- Защита от спама и накрутки лайков.

### Согласованность данных

- Использование идемпотентности и дедупликации событий.
- Приемлема eventual consistency, где это возможно.
