# Результаты сравнения производительности ClickHouse vs Vertica

## Методология тестирования

### Конфигурация тестового окружения:
- **Объем данных**: 100,000, 3,000,000, 5,000,000 и 10,000,000 записей
- **Распараллеливание**: 4-6 процессов
- **Размеры батчей**: 1,000, 5,000, 10,000, 25,000, 50,000, 100,000 записей
- **Платформа**: Docker containers
- **Версии**: ClickHouse 25.3.3.42, Vertica Analytic Database v24.1.0-0

### Используемые скрипты:
- `clickhouse_test.py` - тестирование ClickHouse
- `vertica_test.py` - тестирование Vertica  
- `perfomance_comparison.py` - запуск всех тестов и сбор метрик

## Основные результаты тестирования

### 1. Производительность вставки данных

#### Малые объемы данных (100,000 записей)

**100,000 записей, batch 1,000:**
- ClickHouse: 4.60 сек (21,753 записей/сек)
- Vertica: 8.63 сек (11,583 записей/сек)
- **ClickHouse быстрее в 1.88 раза**

**100,000 записей, batch 5,000:**
- ClickHouse: 3.83 сек (26,133 записей/сек)
- Vertica: 5.73 сек (17,465 записей/сек)
- **ClickHouse быстрее в 1.50 раза**

#### Средние объемы данных (3,000,000 записей)

**3,000,000 записей, batch 10,000:**

- Vertica: 218.80 сек (13,711 записей/сек)
- **ClickHouse быстрее в 1.66 раза**

**3,000,000 записей, batch 25,000:**
- ClickHouse: 137.53 сек (21,813 записей/сек)
- Vertica: 204.41 сек (14,676 записей/сек)
- **ClickHouse быстрее в 1.49 раза**

#### Большие объемы данных (5,000,000+ записей)

**5,000,000 записей, batch 50,000:**
- ClickHouse: 209.27 сек (23,892 записей/сек)

**10,000,000 записей, batch 100,000:**

- Vertica: 575.38 сек (17,380 записей/сек)

## Ключевые выводы

1. **Стабильная производительность вставки**: ClickHouse демонстрирует стабильное превосходство в 1.37-1.88 раза на всех объемах данных
2. **Превосходство в запросах**: ClickHouse показывает значительно лучшую производительность запросов (в 4.77-18.65 раза быстрее)
3. **Масштабируемость**: 
   - ClickHouse поддерживает стабильную производительность ~22-24K записей/сек на всех объемах
   - Vertica показывает производительность ~13-17K записей/сек с некоторой деградацией на средних объемах
4. **Время отклика запросов**: Все запросы в ClickHouse выполняются значительно быстрее требуемых 10 секунд
5. **Критическое различие в сложных запросах**: На больших объемах данных разница в производительности запросов достигает 18.65 раза

### Схемы хранения:

**ClickHouse:**
- Движок: MergeTree
- Партиционирование: по месяцам (toYYYYMM)
- Сортировка: по timestamp, user_id, content_id, event_type

**Vertica:**
- Сегментирование: HASH(user_id) ALL NODES
- Партиционирование: по месяцам (DATE_TRUNC)
- Сортировка: по timestamp, user_id, content_id, event_type

## Итоговое заключение

Для задачи сохранения поведения пользователей был выбран **ClickHouse** по следующим причинам:

1. **Стабильная высокая производительность вставки** - ключевой фактор для real-time аналитики
2. **Производительность запросов** - во всех тестах ClickHouse показал лучшие результаты
3. **Масштабируемость** - производительность остается стабильно высокой при росте объемов данных от 100K до 10M записей
4. **Предсказуемое время отклика** - все запросы выполняются в приемлемые сроки даже на больших объемах
5. **Экономическая эффективность** - open source решение без лицензионных ограничений
